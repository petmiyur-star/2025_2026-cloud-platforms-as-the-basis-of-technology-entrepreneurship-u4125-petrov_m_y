# Лабораторная работа №4: Разработка инфраструктуры MVP AI приложения

## Тема приложения
**«Генерация изображений по текстовому описанию»**  
Пользователь вводит текстовый запрос (например, «кот в скафандре на Луне»), а нейросеть создаёт уникальное изображение. Приложение подходит для дизайнеров, маркетологов и креативных пользователей.

## Цель работы
Спроектировать инфраструктуру для MVP AI‑приложения, выбрать оптимальные облачные сервисы Google Cloud, рассчитать экономическую модель для трёх этапов роста (прототип → тестирование с партнерами → промышленная эксплуатация).

## Схема инфраструктуры
![Архитектура MVP](infrastructure.png)
*Рисунок 1. Архитектура MVP «Генерация по описанию» на Google Cloud*

Компоненты на схеме:
- Пользователь → Cloud CDN → Cloud Load Balancing → Cloud Run (бэкенд)
- Cloud Run вызывает Vertex AI (Imagen) для генерации
- Результат сохраняется в Cloud Storage
- Метаданные запроса – в Firestore
- При повторении запроса CDN отдаёт кэшированное изображение

## Обоснование выбора облачных сервисов

| Сервис | Зачем нужен | Почему именно он |
|--------|-------------|------------------|
| **Cloud Run** | Бэкенд на Python/FastAPI, обработка запросов | Масштабируется до нуля – платим только за реальные вызовы. Для MVP это оптимально. |
| **Vertex AI + Imagen 3** | Генерация картинок по тексту | Готовая модель, не нужно обучать. Плата за каждую генерацию, без простоя. |
| **Cloud Storage** | Хранение сгенерированных изображений | Дешёвое объектное хранилище. Интегрируется с CDN. |
| **Firestore** | База данных для истории запросов, метаданных | NoSQL, легковесная, бесплатный пакет на начальном этапе. Идеально для MVP. |
| **Cloud CDN** | Кэширование популярных результатов | Ускоряет выдачу и уменьшает количество вызовов генерации → экономия. |
| **Cloud Load Balancing** | Распределение трафика и SSL-терминация | Единая точка входа, отказоустойчивость. На старте не обязателен, но нужен для роста. |

**Ключевое решение:** отказ от Compute Engine (виртуальных машин) в пользу Cloud Run позволяет не платить за время простоя, что критично для MVP.

## Экономическая модель

Расчёт выполнен для региона `europe-west1` с использованием GCP Pricing Calculator (2026). Учтены бесплатные квоты.

### Этап 1: Прототип (начальный уровень)
- **Аудитория**: сам разработчик, единичные тесты (≈20 запросов/мес)
- **Общая стоимость**: **≈ $4.15** в месяц (в основном генерация)

| Сервис | Объём | Расчёт | Стоимость |
|--------|-------|--------|------------|
| Cloud Run | 1 млн запросов (в рамках бесплатных квот) | 0 | **$0** |
| Cloud Storage | 5 ГБ | 5 × $0.02 = $0.10 | **$0** (бесплатно до 5 ГБ) |
| Firestore | 50 тыс. чтений/день | 0 | **$0** |
| Vertex AI (Imagen 3) | 100 генераций | 100 × $0.04 = $4.00 | **$4.00** |
| Cloud CDN | 5 ГБ трафика | 5 × $0.03 = $0.15 | **$0.15** |
| **Итого** | | | **≈ $4.15** |

### Этап 2: Тестирование с партнерами
- **Аудитория**: 10–50 партнеров, ≈5 000 запросов/мес (20% из них уникальные)
- **Общая стоимость**: **≈ $182** в месяц

| Сервис | Объём | Расчёт | Стоимость |
|--------|-------|--------|------------|
| Cloud Run | 2 млн CPU-сек, 4 млн GiB-сек | в рамках бесплатных квот | **$0** |
| Cloud Storage | 20 ГБ | 20 × $0.02 = $0.40 | **$0.40** |
| Firestore | 150 тыс. чтений, 50 тыс. записей | (150k/100k)*$0.108 + (50k/100k)*$0.54 ≈ $0.43 | **$0.43** |
| Vertex AI (Imagen 3) | 4 000 генераций | 4 000 × $0.04 = $160.00 | **$160.00** |
| Cloud CDN | 100 ГБ кэшированного трафика | 100 × $0.03 = $3.00 | **$3.00** |
| Cloud Load Balancing | 1 правило, региональный | $18.00 | **$18.00** |
| **Итого** | | | **≈ $181.83** |

### Этап 3: Промышленная эксплуатация (продакшн)
- **Аудитория**: 150 000 активных пользователей, ≈15 млн запросов/мес (40% уникальных)
- **Общая стоимость**: **≈ $1 700** в месяц

| Сервис | Объём | Стоимость (мес) |
|--------|-------|------------------|
| Cloud Run | 100 млн CPU-сек, 200 млн GiB-сек | $180 |
| Cloud Storage | 5 000 ГБ | $100 |
| Firestore | 15 млн чтений, 5 млн записей | $25 |
| Vertex AI (Imagen 3) | 30 тыс. уникальных генераций (1 000/день) × $0.04 | $1 200 |
| Cloud CDN | 2 500 ГБ трафика | $120 |
| Cloud Load Balancing | premium tier | $75 |
| **Итого** | | **≈ $1 700** |

> *Примечание:* при росте числа уникальных запросов затраты на генерацию линейно растут. Для снижения расходов рекомендуется кэширование и переход на более дешёвые модели (например, Imagen 3 Fast).

## Выводы
Спроектирована инфраструктура для MVP AI‑приложения на Google Cloud. Ключевые решения:
- **Серверлесс (Cloud Run)** – позволяет стартовать с нулевыми затратами и легко масштабироваться.
- **Готовые AI‑модели (Vertex AI + Imagen)** – ускоряют разработку и не требуют дорогого обучения.
- **Кэширование и CDN** – снижают количество вызовов генерации, экономят бюджет.
- **Поэтапный рост** – от ≈$4 на прототипе до ≈$1 700 на масштабе в 150 тыс. пользователей.

Рекомендация: на начальном этапе активно использовать бесплатные квоты, на этапе партнёрского тестирования – добавить мониторинг и алерты, перед выходом в продакшн – зафиксировать скидки на генерации через резервирование. Предложенная архитектура является экономически обоснованной и технологически реализуемой.
